## 3. JSON 参数说明

区域 5 通过两个文本框描述道路图：**点集**定义路口/转角位置，**边集**定义哪些点之间铺道路。点击「应用布局」后，插件会解析 JSON、重建 `ICity Base` 网格，再触发建筑/道路刷新。

### 3.1 基本结构

| 输入框 | 含义 | 最少要求 |
|--------|------|----------|
| **点集 JSON** | 道路网络中的顶点坐标 | ≥ 2 个点 |
| **边集 JSON** | 连接两点的道路段（用点索引表示） | ≥ 1 条边 |

- 坐标使用 **Blender 世界坐标**（单位：米）；`z` 可省略，默认 `0`。
- 边集索引 **从 0 开始**，对应点集数组下标：`[0, 1]` 表示第 0 个点连到第 1 个点。
- 支持标准 **JSON**；也支持 Python 风格字面量（如 `[(0,0), (10,0)]`，由 `ast.literal_eval` 解析）。
- 重复边会自动去重；自环边（起点=终点）会报错。

### 3.2 点集格式

每个点可以是 **数组** 或 **对象**：

```json
// 写法 A：数组（推荐，与「载入示例」一致）
[
  [-216, -167, 0],
  [216, -167, 0],
  [216, 167, 0],
  [-216, 167, 0]
]

// 写法 B：对象
[
  {"x": -216, "y": -167, "z": 0},
  {"x": 216, "y": -167},
  {"x": 216, "y": 167},
  {"x": -216, "y": 167}
]
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `x` / `[0]` | 数字 | 东西方向（Blender X 轴） |
| `y` / `[1]` | 数字 | 南北方向（Blender Y 轴） |
| `z` / `[2]` | 数字，可选 | 高度，省略时为 `0` |

### 3.3 边集格式

每条边连接点集中的两个索引：

```json
[
  [0, 1],
  [1, 2],
  [2, 3],
  [3, 0]
]
```

也支持对象写法：`{"a": 0, "b": 1}` 或 `{"start": 0, "end": 1}`。

**单块矩形示例（游标在原点时）的对应关系**：

```text
  点3 (-216, 167) ──────── 点2 (216, 167)
       │                        │
       │      1 个街区面         │
       │                        │
  点0 (-216,-167) ──────── 点1 (216,-167)

  边：0→1（南）  1→2（东）  2→3（北）  3→0（西）
```

### 3.4 合并写法（可选）

点集与边集可写在同一个 JSON 对象里；此时边集输入框可留空：

```json
{
  "points": [
    [-216, -167, 0],
    [216, -167, 0],
    [216, 167, 0],
    [-216, 167, 0]
  ],
  "edges": [
    [0, 1],
    [1, 2],
    [2, 3],
    [3, 0]
  ]
}
```

若合并对象里已有 `edges`，且边集输入框也填写了内容，则以**边集输入框**为准。

### 3.5 与面板选项的关系

| 选项 | 作用 | 建议 |
|------|------|------|
| **以游标为布局中心** | 应用布局前，将点集几何中心平移到 3D 游标（仅 XY） | 默认开启；与「生成基础城市」视觉居中一致 |
| **自动补边界面** | 在凸包上自动补闭合边界面 | 单块矩形示例会自动关闭；手写闭合矩形时也建议关闭 |

「载入示例」会填入 **432×334m** 整数矩形（若已生成城市则按当前包围盒取整），并将矩形中心对齐游标。

### 3.6 应用后 Info 字段

成功应用后，Info / `scg_layout_status` 会报告：

- **N 个点**：点集顶点数  
- **M 条道路边**：去重后的边数  
- **K 个街区面（预期 K）**：插件识别出的可建街区面数；单块矩形应为 `预期 1`  
- **边界面=开启/关闭**：是否启用了自动补边界面  

### 3.7 常见注意点

1. **索引越界**：边 `[2, 5]` 要求点集至少有 6 个点（索引 0～5）。  
2. **未闭合道路**：边集未形成闭环时，可能无法建面或出现裸道路；单块城市需 4 点 4 边闭合。  
3. **试验后恢复**：自定义布局不稳定时，点 **恢复基础城市** 可还原默认 `ICity Base` 并按区域 1 参数重新生成。  
4. **草图提取**：从图像提取的结果也会写回同一套点集/边集 JSON 格式，可手动微调后再应用。

## 4. 核心流程（`apply_layout_graph`）

```text
parse_layout_inputs（JSON / Python 列表）
  → Edit Mode BMesh：清空 ICity Base 几何
  → 按点/边建 mesh + 自动建面（半边遍历 + 可选凸包边界）
  → Object Mode：写面/边 Attribute（space type、Road del、Tree/Bench/Light…）
  → 保留部分 ICity 偏移/街道 Attribute 默认值
  → set_road_texture + Road 2 节点 Tree/Bench/Light
  → sync_city / 程序化建筑刷新
```

**输入格式**：见上文 **§3 JSON 参数说明**。

## 5. 草图提取（`extract_layout_from_image`）

1. 加载图像 → 降采样 mask（暗像素 = 道路）  
2. 识别节点（度数 ≠ 2 或共线拐点）+ 追踪线段  
3. 映射到世界坐标（居中 + `scg_layout_world_scale`）  
4. 写回点集/边集 JSON；可选立即 `apply_layout_graph`  

**局限（初版，待微调）**：依赖高对比度线稿；复杂草图可能漏边/多节点；阈值与分辨率需手动调节。

### 问题与修复（规整街区示例「巳」形，历史记录）

**原因**：开启「自动补边界面」时，凸包会添加 **0→2、2→8** 等弦边（跳过中间格点），再配合 `contextual_create`  fallback，ICity 只正确生成一块街区，其余道路无建筑。

**修复**（`layout_control.py`）：

- 凸包弦边若共线经过其它输入点 → **不添加**
- 只保留 **内部不含其它顶点** 的最小面（4 个网格单元）
- 统一 CCW 绕序；去掉 `contextual_create` 误合并

**验证**：载入示例 → 应用布局 → Info 应含 `预期 4` 个街区面；视口为 **2×2 四块城市**，无外围「巳」形裸道路。
