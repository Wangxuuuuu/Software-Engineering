# Day 20–21 手动渲染操作指南（A10）

> **目标**：不依赖插件「渲染 Panel」，在 Blender 4.1 中手动输出 **PNG 静帧** + **MP4 动画**，满足主文档 §8.1.1 与验收 **A10**。  
> **建议输出目录**：项目内 `团队作业/renders/`（或 `.blend` 同级的 `renders/`）

---

## 0. 渲染前：把场景准备到「可拍」

按主文档 §11 演示顺序，建议先做完再渲染（至少包含以下）：

| 步骤 | 操作 | 对应验收 |
|------|------|----------|
| 1 | 启用 ICity 插件 → **Start ICity** | A1 |
| 2 | Smart City → **生成基础城市** | A2 |
| 3 | （可选）应用模板 0 或 1、切换道路纹理 | A5/A3 |
| 4 | （可选）添加路灯 | A4 |
| 5 | 区域 6 → **添加车辆与行人**；**添加中央山水**；**添加船只** | A7/A8 |
| 6 | （可选）区域 5 应用布局示例 | A9 |
| 7 | （可选）区域 7 NL「将天色变暗」做夜景静帧 | A6 |

**保存工程**：`文件 → 另存为` → 如 `icity_demo.blend`（渲染输出路径常相对此文件）。

**视口**：右上角着色方式选 **材质预览** 或 **渲染**（与日常验收一致）。

在资源管理器中创建（或在 Blender 文件浏览器中新建）：


后续所有成片放这里，便于交给前端同学。

## 2. 设置渲染引擎与画质（一次性）

菜单：**编辑 → 偏好设置 → 系统 → 计算设备**

- 若有 NVIDIA 显卡：Cycles 可勾选 **CUDA/OptiX**（渲静帧更快）；没有独显可跳过。

### 2.1 推荐：静帧用 EEVEE（快）

1. 右侧 **属性编辑器**（快捷键 `N` 可开关侧栏，渲染属性在右侧竖排图标）
2. 点击 **渲染属性**（相机图标）
3. **渲染引擎** → **EEVEE**
4. 展开 **采样**：
   - **视口采样** / **渲染采样**：`64` 或 `128`（越高越干净，越慢）
5. 勾选 **辉光**（Bloom）可选，夜景略好看

### 2.2 可选：高质量静帧用 Cycles

1. **渲染引擎** → **Cycles**
2. **设备** → GPU Compute（有显卡时）
3. **采样 → 渲染** → `128`～`256`（演示用 128 即可，全片可等）

**动画视频**建议仍用 **EEVEE**，1–120 帧出片快。

---

## 3. 设置输出路径与分辨率

右侧 **输出属性**（打印机图标）：

| 项 | 静帧 PNG 建议 | 动画 MP4 建议 |
|----|---------------|---------------|
| **分辨率 X × Y** | `1920 × 1080` | 同左 |
| **分辨率 %** | `100%` | `100%` |
| **帧率** | — | `24` 或 `30` |
| **帧范围** | 单帧即可 | `1` ~ `120`（与 `scg_animation_frame_end` 一致） |

### 3.1 静帧 PNG

- **输出** → 点击文件夹图标
- 路径填（相对 blend 文件）：
  ```text
  //../团队作业/renders/city_preview.png
  ```
  或绝对路径：
  ```text
  d:\Blender\4.1\scripts\addons\icity\团队作业\renders\city_preview.png
  ```
- **文件格式** → **PNG**
- **颜色** → **RGBA**（带透明可选；城市全景用 RGB 即可）

### 3.2 动画 MP4

- **输出** 路径：
  ```text
  //../团队作业/renders/city_animation.mp4
  ```
- **文件格式** → **FFmpeg 视频**
- **编码** → **MPEG-4**（或 H.264）
- **输出质量** → **高** 或 **无损**（演示用「高」即可）

> 若列表里没有 FFmpeg：需在安装 Blender 时勾选 FFmpeg，或改用下方「PNG 序列 + 外部合成」。

## 4. 摆放相机（setup_camera 的手动版）

### 4.1 添加相机

1. 视口按数字键 **`0`**（小键盘）可切到「当前相机视角」；若提示无相机：
2. **`Shift + A` → 相机 → 相机**
3. 选中相机，按 **`Ctrl + Alt + Numpad 0`**（或菜单 **视图 → 对齐视图 → 对齐活动相机到视图**）

### 4.2 推荐机位（城市全景）

在 **正视图/透视** 下用 **`G` 移动、`R` 旋转** 调整相机，或用 **视图 gizmo**：

| 镜头 | 位置建议 | 用途 |
|------|----------|------|
| **鸟瞰** | 城市斜上方，俯视约 35°–50° | 主文档 §11 步骤 2「场景全景」、A10 主图 |
| **街景** | 贴近道路、高度约 2m | 车辆动画、路灯特写 |
| **湖面** | 有山水时，低角度对准水面与船 | A8 生态 |

**焦距**：选中相机 → 物体数据属性（绿色相机图标）→ **焦距** `35`–`50` mm（越大越「长焦」、建筑更整齐）。

**景深**（可选）：同页勾选 **景深**，焦点对准城市中心。

### 4.3 锁定机位

满意后：**物体 → 变换 → 锁定位置/旋转**（或仅记笔记，避免误拖）。

### 4.4 常见问题：视口好看，但小键盘 0 只剩「中间一块」

**原因**：`Ctrl+Alt+0` 只同步相机位置/朝向；**视口视野 ≠ 相机焦距**。按 **0** 后看到的是真实渲染画幅（默认焦距约 50mm，偏窄）。

**处理（相机位置不变，扩大范围）**：

1. 选中 **Camera** → 物体数据 → **焦距** 改小（视野变宽）：先试 **24mm**，仍窄用 **18mm**
2. 在 **小键盘 0** 下边调边看，以相机画面为准（与 **F12** 一致）
3. 鸟瞰俯视图可改 **正交相机**，调大 **正交比例**

**更稳流程**：先摆相机大致位置 → **小键盘 0** → 只调焦距，少用「先调视口再 Ctrl+Alt+0」。

### 4.5 常见问题：对齐相机后，不按 0 时滚轮无法继续放大

**原因**：缩放到 **相机物体** 附近会顶住；或误开 **相机到视图** 锁定。

**处理**：

- 大纲里 **隐藏相机**（眼睛图标），或选中相机 **`H`** 隐藏后再缩放
- **`小键盘 .`** 先聚焦到城市/地面，再滚轮缩放
- 相机视角下 **`N` → 视图 → 取消「相机到视图」**

## 5. 渲染静帧图片（A10 · PNG）

1. 时间轴停在 **第 1 帧**（或夜景某一帧）
2. 视口切到相机视角：**小键盘 `0`**
3. 菜单 **渲染 → 渲染图像**（或 **`F12`**）
4. 等待完成后，图像编辑器窗口显示结果
5. **图像 → 另存为**（或渲染前已设输出路径则自动写入 `city_preview.png`）

**建议多拍几张**（交前端 / 文档）：

| 文件名 | 场景状态 |
|--------|----------|
| `city_preview_day.png` | 白天 + 基础城市 |
| `city_preview_night.png` | NL「将天色变暗」后 |
| `city_ecology.png` | 山水 + 船只 |
| `city_layout.png` | 应用布局后 |
| `city_nl_demo.png` | 车辆 + 路灯等 |

每张 **`F12`** 前改 **输出** 文件名即可。

## 6. 渲染动画视频（A10 · MP4）

### 6.1 确认动画可播

1. 时间轴范围：**1 – 120**（与区域 6「动画结束帧」一致）
2. 按 **`Space`** 在视口播放：车/人/船应沿路径移动
3. 若无动画：先 **区域 6 → 添加车辆与行人**（及船只）

### 6.2 用 EEVEE 渲 MP4

1. **渲染引擎** → EEVEE
2. **输出属性** → 格式 **FFmpeg 视频**，路径 `city_animation.mp4`
3. **帧范围** → 起始 `1`，结束 `120`
4. 相机视角：**小键盘 `0`**
5. 菜单 **渲染 → 渲染动画**（或 **`Ctrl + F12`**）
6. 等待（EEVEE 1080p 约数分钟级，视机器而定）
7. 用 **电影和电视 / VLC** 打开 `city_animation.mp4` 检查

### 6.3 备选：PNG 序列（FFmpeg 不可用时）

1. **文件格式** → **PNG**
2. 输出路径：`//../团队作业/renders/frames/frame_####.png`
3. **`Ctrl + F12`** 渲染动画
4. 用 DaVinci / Premiere / `ffmpeg` 合成 MP4：
   ```bash
   ffmpeg -framerate 24 -i frame_%04d.png -c:v libx264 -pix_fmt yuv420p city_animation.mp4
   ```

## 9. 常见问题

| 现象 | 处理 |
|------|------|
| 渲染全黑 | 检查是否「将天色变暗」过；加 **日光** 或切白天模板 |
| 材质发粉/发灰 | 视口用 **渲染**；EEVEE 开 **屏幕空间反射** |
| 只有灰模 | 未 Start ICity 或未生成城市 |
| 动画不动 | 先添加动态元素；帧范围是否 1–120 |
| MP4 无法打开 | 改 H.264；或改用 PNG 序列 + ffmpeg |
| 输出路径找不到 | 用 **绝对路径**；渲染后在 **输出属性** 点路径旁的文件夹图标打开 |
