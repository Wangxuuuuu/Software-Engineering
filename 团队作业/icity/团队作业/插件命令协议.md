# Smart City Generator — 插件命令协议（NL / LLM 联调）

> **版本**：P2（2026-05-23）  
> **用途**：LLM 模块、离线规则引擎、区域 7「执行指令」均输出同一 JSON，由 `core/command_executor.py` 分发。

---

## 1. 顶层结构

```json
{
  "source": "nl_input",
  "raw_text": "用户原始中文（可选，便于日志）",
  "commands": [
    { "action": "<action_name>", "params": { } }
  ],
  "reply": "一句话中文摘要，供面板显示（可选）"
}
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `commands` | **是** | 有序命令数组；至少 1 条 |
| `action` | **是** | 见 §2 |
| `params` | 否 | 默认 `{}` |
| `reply` | 否 | UI 摘要；缺省时由 executor 从最后一条结果推断 |
| `source` | 否 | 建议 `nl_input` / `llm` / `offline_rule` |
| `raw_text` | 否 | 原始用户输入 |

**一条用户话可对应多条 command**（按数组顺序执行）：

```json
{
  "raw_text": "将天色变暗并添加车辆",
  "commands": [
    { "action": "set_environment", "params": { "mode": "night" } },
    { "action": "add_dynamic_element", "params": { "type": "car" } }
  ],
  "reply": "已切换夜间并添加车辆"
}
```

## 2. Action 列表（P0 已注册）

### 2.1 `apply_template`

按 `config/templates.json` 一键配置并生成城市（同面板「应用模板」）。

```json
{ "action": "apply_template", "params": { "template_id": "0" } }
```

| param | 类型 | 说明 |
|-------|------|------|
| `template_id` | string / int | `"0"`、`"1"` … |

**A6 示例话术**：「应用模板 0」

---

### 2.2 `apply_asset_config`

写入 `scg_tree_type` / `scg_road_type` / `scg_bench_type` 等；默认重新生成城市。

```json
{
  "action": "apply_asset_config",
  "params": {
    "tree_type": 1,
    "road_texture": 1,
    "bench_type": 1,
    "building_density": "medium",
    "regenerate": true
  }
}
```

| param | 类型 | 默认 | 说明 |
|-------|------|------|------|
| `tree_type` | int | — | 1 或 2 |
| `road_texture` | int | — | 1 或 2（写入 `scg_road_type`） |
| `bench_type` | int | — | 1 或 2 |
| `building_density` | string | — | `low` / `medium` / `high` |
| `regenerate` | bool | `true` | 是否调用「生成基础城市」 |

**A6 示例话术**：「将树木、道路和座椅均设为类型 1」

---

### 2.3 `set_road_texture`

仅改道路纹理（同 `scg.apply_road_texture`），不整城重建。

```json
{ "action": "set_road_texture", "params": { "type": 1 } }
```

| param | 类型 | 说明 |
|-------|------|------|
| `type` | int | **1** 或 **2** |

**A6 示例话术**：「将道路改为类型 1」

---

### 2.4 `set_environment`

昼夜 / 雨天（阶段 1 雨天为方案 A：环境变暗，无雨粒子）。

```json
{ "action": "set_environment", "params": { "mode": "night" } }
```

| param | 类型 | 取值 |
|-------|------|------|
| `mode` | string | `day` · `night` / `dark` / `dusk` · `rainy` |

**A6 示例话术**：「将天色变暗」→ `mode: "night"`  
**扩展话术**：「切换为雨天模式」→ `mode: "rainy"`

---

### 2.5 `add_dynamic_element`

添加车辆 / 行人 / 船只（同区域 6 动态元素）。

```json
{ "action": "add_dynamic_element", "params": { "type": "car", "count": 2 } }
```

| param | 类型 | 说明 |
|-------|------|------|
| `type` | string | `car` · `pedestrian` · `boat` |
| `count` | int | 可选；缺省用面板 `scg_*_count` |

- `type: "car"`：**仅添加车辆**（执行时行人数量临时为 0，不随面板默认行人数添加）
- `type: "pedestrian"`：**仅添加行人**（执行时车辆数量临时为 0）

**A6 示例话术**：「添加 2 辆车辆」→ `type: "car"`, `count: 2`（仅车辆，不含行人）

---

### 2.6 `add_street_lights`

追加团队路灯（同 `scg.apply_street_lights`）。P3 扩展，P0 已注册。

```json
{ "action": "add_street_lights", "params": { "night_emission": false } }
```

## 3. A6 验收五条 → JSON 样例

| # | 用户输入 | commands |
|---|----------|----------|
| 1 | 将天色变暗 | `[{"action":"set_environment","params":{"mode":"night"}}]` |
| 2 | 将道路改为类型 1 | `[{"action":"set_road_texture","params":{"type":1}}]` |
| 3 | 应用模板 0 | `[{"action":"apply_template","params":{"template_id":"0"}}]` |
| 4 | 将树木、道路和座椅均设为类型 1 | `[{"action":"apply_asset_config","params":{"tree_type":1,"road_texture":1,"bench_type":1}}]` |
| 5 | 添加 2 辆车辆 | `[{"action":"add_dynamic_element","params":{"type":"car","count":2}}]` |

说明：`type: "car"` 时**仅添加车辆**，临时将行人数量置 0，不读取面板默认行人数。

---

## 4. 错误约定

- 未注册 `action` → executor 返回 `✗ 未注册 action: …`
- 未 Start ICity → `✗ …：请先 Start ICity 并生成基础城市`
- 单条失败不中断后续 command（P0 行为）；全部成功与否由返回元组第一项表示

---

## 5. 代码入口

```python
from icity.core.command_executor import execute_commands

payload = {
    "commands": [{"action": "set_environment", "params": {"mode": "night"}}],
}
ok, reply, lines = execute_commands(bpy.context, payload)
```

Handler 实现：`core/nl_handlers.py`  
注册时机：`scg_register.register()` → `register_default_commands()`

---

## 6. 与 templates.json 的关系

- **静态模板表**：`config/templates.json`（插件维护，LLM **不修改文件**）
- **NL / LLM**：只产出 `apply_template` 或 `apply_asset_config` 等 **commands**
- 模板字段含义见 [`Day 9/templates与组内协作说明.md`](Day%209/templates与组内协作说明.md)
