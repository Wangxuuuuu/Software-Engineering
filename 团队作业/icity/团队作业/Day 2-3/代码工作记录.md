## 3. 各文件说明（学习用）

### 3.1 `scg_properties.py`（新增）

**作用**：向 `bpy.types.Scene` 注册团队面板用的枚举 / 布尔属性。

| 属性名 | 类型 | 含义 |
|--------|------|------|
| `scg_city_scale` | Enum | 小型 `SMALL` / 中型 `MEDIUM` |
| `scg_building_density` | Enum | 低 `LOW` / 中 `MEDIUM` / 高 `HIGH` |
| `scg_building_type` | Enum | 建筑样式 类型1 / 类型2 |
| `scg_road_type` | Enum | 道路类型 类型1 / 类型2 |
| `scg_tree_type` | Enum | 树木类型 类型1 / 类型2 |
| `scg_bench_type` | Enum | 座椅类型 类型1 / 类型2 |
| `scg_enter_edit_after_generate` | Bool | 生成后是否进入 Edit（默认 `False`） |

**注册时机**：`scg_register.register()` → `scg_properties.register()`。

---

### 3.2 `scg_register.py`（新增）

**作用**：把 SCG 相关注册集中在一处，避免 `__init__.py` 继续膨胀。

```text
register() 顺序：
  1. scg_properties.register()   # Scene 属性
  2. operators.register()      # scg.* Operator
  3. ui.register()             # Panel

unregister() 顺序（相反）：
  ui → operators → scg_properties
```

---

### 3.3 `ui/panel_smart_city.py`（新增）

**作用**：3D 视图侧边栏 **ICity** 分类下、**bl_order=5** 的独立面板。

| 类名 | `bl_idname` | 说明 |
|------|-------------|------|
| `SCG_PT_smart_city_generator` | `SCG_PT_smart_city_generator` | 主面板 |
| `SCG_OT_placeholder_info` | `scg.placeholder_info` | 「检查模块状态」提示 |

**UI 分区**：

1. 场景状态（是否已 Start）
2. ICity 基础：未 Start 仅 **Start**；已 Start 显示 **Edit City**（`scg.edit_city`）
3. **1. 基础场景生成**：六项 `scene.scg_*` 属性 + **生成基础城市**
4. 2–6 占位说明（阶段二～五）

**与 ICity editor 分工**：本面板 **不含** City·Road、资产预览图（仍在 `SNA_PT_ICITY_EDITOR`）。

### 3.6 `core/city_generator.py`（Day 1 占位 → Day 2–3 核心）

**主要函数**：

| 函数 | 作用 |
|------|------|
| `icity_scene_ready()` | 是否已有 ICity 集合 + ICity Base |
| `ensure_icity_started()` | 无场景则 `sna.start_5209e` |
| `_ensure_object_mode_icity_base()` | 强制 Object 模式再写 Attribute |
| `enter_icity_edit_mode()` | 只进入 Edit，不 toggle |
| `apply_mesh_city_attributes()` | 按规模/密度写面 Attribute（space type、楼层等） |
| `apply_mesh_road_attributes()` | 写边 Attribute（Road del、Light、Tree、Bench） |
| `_apply_road_node_object()` | 设置 `Road 2` 节点上的 Object 引用 |
| `_apply_road_texture()` | 团队道路贴图 → 材质（文件存在时） |
| `_refresh_procedural_buildings()` | `procedural_building_filter` + `park_filter` + `sync_city` |
| `_flush_viewport_refresh()` | `update_tag` + depsgraph + 3D 视口重绘 |
| `generate_base_city_from_scene()` | **一键生成入口** |

**与 ICity 的关系**：通过 `bpy.ops.sna.*` 和直接写 `mesh.attributes`，不复制 ICity 三千行 UI 代码。

## 4. 调用流程（生成基础城市）

```text
用户点击「生成基础城市」
    │
    ▼
SCG_OT_generate_base_city.execute()
    │
    ▼
generate_base_city_from_scene(scene, context)
    │
    ├─ ensure_icity_started()          → bpy.ops.sna.start_5209e (若需要)
    ├─ _ensure_object_mode_icity_base()
    ├─ apply_mesh_city_attributes()      → 面：space type / 楼层 / Procedural index
    ├─ apply_mesh_road_attributes()      → 边：Road del / Light / Tree / Bench
    ├─ _apply_road_texture()             → Road 2 材质（有贴图时）
    ├─ _apply_road_node_object()         → Light / Tree / Bench 物体
    ├─ _sync_icity_scene_props_for_ui()  → 同步 ICity editor 枚举
    ├─ _flush_viewport_refresh()
    ├─ _refresh_procedural_buildings()   → filter + sync
    └─ _flush_viewport_refresh()
    │
    ▼
（若勾选 scg_enter_edit_after_generate）
enter_icity_edit_mode()                  → 不 toggle
```
