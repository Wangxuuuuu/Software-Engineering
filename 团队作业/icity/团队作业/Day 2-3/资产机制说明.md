# Day 2-3：资产机制说明

> 说明树木/座椅/道路贴图等资产 **从哪来**、**如何与面板「类型1/类型2」对应**、与 ICity 原资产浏览器的关系，以及 **阶段二扩充** 时如何接到现有代码。  
> 对应代码：`core/assets_manager.py`、`core/city_generator.py`（`_object_from_collection`、`_apply_road_texture`）。

---

## 1. 总览：两套资产选取机制

ICity 插件里与街道/城市相关的资产，存在 **两套并行机制**：

| 机制 | 使用者 | 选取方式 | 典型入口 |
|------|--------|----------|----------|
| **A. Smart City（团队）** | `scg.generate_base_city` | 集合内 **Object 序号**（0/1） | 面板「树木类型1/2」 |
| **B. ICity 原编辑器** | `sna.road_apply_5c3ab` 等 | **按 Object 名称** + 缩略图浏览器 | Road → Bench → `Bench10_Bench_ICity_Default` |

两者最终都写入同一处：**几何节点组 `Road 2`** 中 `Tree` / `Bench` / `Light` 等节点的 **Object 输入**（一般为 `inputs[4]`）。

```text
面板 / 浏览器选择
    │
    ├─ A：_object_from_collection("ICity_Tree", tree_idx)
    └─ B：bpy.data.objects[sna_street_asset_browser]
    │
    ▼
Road 2.nodes["Tree"].inputs[4].default_value = <Object>
    │
    ▼
ICity Road 几何节点沿路实例化该模型
```

## 2. 资产文件在哪里？

### 2.1 启动模板（Day 2-3 树木/座椅的主要来源）

| 路径 | 作用 |
|------|------|
| `assets/ICity start.blend` | 点击 **Start**（`sna.start_5209e`）时整包 `append` 进当前场景 |

Start 后会得到分类集合（节选）：

| 集合名 | 内容 |
|--------|------|
| `ICity_Tree` | 树木模板 Object |
| `ICity_Bench` | 长椅模板 Object |
| `ICity_Light` | 路灯模板 Object |
| `ICity Assets` | 隐藏总库（视口/渲染隐藏） |
| `ICity_Procedural` | 程序化建筑实例 |
| `ICity_Park` | 公园 |

**Day 2-3 的「类型1/类型2」** 默认从上述集合里 **按序号** 取 Object，**不** 直接读 `assets/team/` 下的 `.blend`。

### 2.2 ICity 默认资产库（原面板 Bench10、Tree9 等）

| 路径 | 作用 |
|------|------|
| `assets/Assets/Default/Road/Road assets.blend` | 大量街道物件（Bench1～11、Tree1～14、路灯、标志等） |
| `assets/Assets/Default/Procedural/` | 程序化建筑 `.blend` |
| `assets/Assets/Default/Park/` | 公园 `.blend` |
| `assets/Assets/Default/Building presets/` | 预设建筑 |
| `assets/Assets/Default/All assets` | **文本索引**：`['资产名', '显示名', 'blend路径', 图标索引]` |
| `assets/Assets/Default/All materials` | 材质索引 |

索引示例（节选）：

```text
['Bench10_Bench_ICity_Default', 'Bench10_Bench_ICity_Default',
 '.../Road/Road assets.blend', 0]
['Tree9_Tree_ICity_Default', 'Tree9_Tree_ICity_Default',
 '.../Road/Road assets.blend', 0]
```

插件 **Refresh / Read** 会扫描 `assets/Assets/Default/` 下各主题子目录的 `.blend`，填充内存列表 `append['sna_all_assets']`，再按名称过滤出道路资产浏览器列表。

> 说明：仓库内 `All assets` 里部分路径可能仍指向作者本机（如 `G:\My Drive\...`）；运行时 ICity 会尝试替换为插件本地 `assets/Assets` 路径。若缩略图或 append 失败，需在 Blender 内执行一次资产 Refresh。

### 2.3 团队资产目录（阶段二重点）

| 路径 | 状态（Day 2-3） | 用途 |
|------|-----------------|------|
| `assets/team/textures/road_type1.jpg` | 待放入 | `scg_road_type` → `_apply_road_texture` |
| `assets/team/textures/road_type2.jpg` | 待放入 | 同上 |
| `assets/team/models/street_light.blend` | 待放入 | 阶段二路灯（`STREET_LIGHT_BLEND` 已预留路径） |

映射定义见 `core/assets_manager.py`。

## 3. Smart City：「类型1 / 类型2」如何对应？

### 3.1 面板 → 代码

`scg_properties.py` 中树木/座椅/建筑/道路等均为：

```python
_SCG_TYPE_ITEMS = (
    ("1", "类型1", "资产样式 1", 0),
    ("2", "类型2", "资产样式 2", 1),
)
```

`city_generator._type_enum_to_index`：`"1" → 0`，`"2" → 1`。

### 3.2 树木 / 座椅（3D Object）

```python
# core/city_generator.py（逻辑摘要）
tree_idx = _type_enum_to_index(scene.scg_tree_type)   # 0 或 1
bench_idx = _type_enum_to_index(scene.scg_bench_type)

_apply_road_node_object("Tree",  _object_from_collection("ICity_Tree",  tree_idx))
_apply_road_node_object("Bench", _object_from_collection("ICity_Bench", bench_idx))
```

`_object_from_collection`：**不查名字**，只取 `collection.objects[索引]`（越界则夹到最后一项）。

| 面板 | 索引 | 实际物体 |
|------|------|----------|
| 树木 类型1 | 0 | `ICity_Tree.objects[0]` |
| 树木 类型2 | 1 | `ICity_Tree.objects[1]` |
| 座椅 类型1 | 0 | `ICity_Bench.objects[0]` |
| 座椅 类型2 | 1 | `ICity_Bench.objects[1]` |

路灯在 Day 2-3 中固定取 `ICity_Light` 的第 0 个 Object（未做类型切换）。

### 3.3 边 Attribute（开关）

生成时还会写 `ICity Base` 网格 **边** 上的布尔 Attribute（与 ICity Road 模式一致）：

| Attribute | 生成时默认值 |
|-----------|----------------|
| `Road del` | False（该边为道路） |
| `Tree` | True |
| `Bench` | True |
| `Light` | False |

几何节点根据边 Attribute 决定是否沿路放置对应实例；**具体长什么样** 由 `Road 2` 节点上的 Object 引用决定。

### 3.4 道路类型（2D 贴图）

| 面板 | 代码 | 行为 |
|------|------|------|
| 道路 类型1 | `road_idx = 0` | 尝试加载 `assets/team/textures/road_type1.jpg` |
| 道路 类型2 | `road_idx = 1` | 尝试加载 `road_type2.jpg` |

若文件 **不存在**，`_apply_road_texture` 静默跳过，视口仍用 ICity 默认道路材质。

### 3.5 建筑样式 类型1/2

写入面 Attribute `Procedural index`（0/1），并同步 `scene.sna_procedural_building_browser` 到 `ICity_Procedural` 集合中对应序号的建筑名——与树木/座椅的「集合序号」思路类似，但作用于 **程序化建筑** 而非 `Road 2` 街道节点。

## 4. 在 Blender 中自查「类型1/2」对应谁

Start 后于 Scripting 运行：

```python
import bpy

for coll_name in ("ICity_Tree", "ICity_Bench", "ICity_Light"):
    coll = bpy.data.collections.get(coll_name)
    if not coll:
        print(coll_name, ": 不存在")
        continue
    for i, obj in enumerate(coll.objects):
        print(f"  [{i}] 类型{i+1} → {obj.name}")
```

建议把输出抄到团队记录里，避免换机器/换 ICity 版本后序号对应关系变化而不知。

## 5. ICity 原编辑器：Bench 图片选择从哪来？

### 5.1 流程

```text
1. 扫描 assets/Assets/Default/**/*.blend
2. 写入 append['sna_all_assets']（及按主题过滤列表）
3. 用户选 Road → Bench（sna_street_asset_type）
4. 浏览器枚举 sna_street_asset_browser（名称含 Bench 的 Object）
5. 若场景中尚无该 Object → wm.append 从 Road assets.blend 追加
6. 移入 ICity_Bench 等集合
7. Road apply → Road 2.nodes["Bench"].inputs[4] = 所选 Object
```

### 5.2 与 Smart City 的差异

| 对比项 | Smart City | ICity 原 Road 面板 |
|--------|------------|-------------------|
| 可选数量 | 固定 2 档（类型1/2） | 索引库中全部 Bench/Tree（十余种+） |
| 识别方式 | 集合 **下标** | Object **名称** |
| 是否 append | 依赖 Start 已带入的 Object | 常按需从 `Road assets.blend` append |
| UI | 枚举下拉 | 带预览图的资产浏览器 |

**本质相同**：都是给 `Road 2` 的节点塞一个 `bpy.types.Object` 引用。

## 6. 当前代码中的资产接口（Day 2-3）

### 6.1 `core/assets_manager.py`

```python
TEAM_ASSETS_ROOT = .../assets/team

ROAD_TEXTURES = {
    1: .../textures/road_type1.jpg,
    2: .../textures/road_type2.jpg,
}
STREET_LIGHT_BLEND = .../models/street_light.blend

def road_texture_path(texture_type: int) -> str: ...
```

### 6.2 `core/city_generator.py` 相关函数

| 函数 | 作用 |
|------|------|
| `_object_from_collection(name, index)` | 从 `ICity_Tree` 等取第 N 个 Object |
| `_apply_road_node_object(street_type, obj)` | 写入 `Road 2.nodes[street_type].inputs[4]` |
| `_apply_road_texture(road_type_index)` | 团队 JPG → 材质 → `Road 2.nodes["Road"].inputs[2]` |
| `apply_mesh_road_attributes(mesh)` | 边 Attribute Tree/Bench/Light 开关 |

### 6.3 生成调用链（资产相关片段）

```text
generate_base_city_from_scene()
  ├─ apply_mesh_road_attributes()          # 边：Tree/Bench=True
  ├─ _apply_road_texture(road_idx)         # 团队道路贴图（有文件才生效）
  ├─ _apply_road_node_object("Tree", ...)  # ICity_Tree[i]
  ├─ _apply_road_node_object("Bench", ...) # ICity_Bench[i]
  └─ _apply_road_node_object("Light", ...) # ICity_Light[0]
```

## 7. 阶段二：如何扩充资产并接到现有接口

### 7.1 道路贴图（2D，优先）

1. 制作或收集 `road_type1.jpg`、`road_type2.jpg`
2. 放入 `assets/team/textures/`
3. 无需改枚举：`scg_road_type` 已映射到 `ROAD_TEXTURES`
4. 验收：切换类型1/2 后道路材质肉眼可辨（待办 A3）

### 7.2 路灯等新 3D 模型

```text
assets/team/models/street_light.blend
  → 阶段二实现 append_if_missing()
  → 对象放入 ICity_Light（或单独集合再引用）
  → _apply_road_node_object("Light", obj)
```

建议在 `assets_manager.py` 增加类似：

```python
STREET_LIGHTS = {
    1: ("street_light.blend", "ObjectNameInBlend"),
    2: ("street_light_alt.blend", "ObjectName2"),
}
```

### 7.3 树木 / 座椅：从「盲索引」升级为「显式映射」（推荐）

当前类型1/2 = 集合第 0/1 个物体，**不稳定**（集合顺序可能随版本变化）。阶段二建议：

**做法 A（推荐）**：在 `assets_manager.py` 写名称表，生成前按需 append

```python
# 示例（名称需与 Road assets.blend 内 Object 名一致）
TREE_OBJECTS = {
    1: "Tree1_Tree_ICity_Default",
    2: "Tree9_Tree_ICity_Default",
}
BENCH_OBJECTS = {
    1: "Bench1_Bench_ICity_Default",
    2: "Bench10_Bench_ICity_Default",
}

def resolve_street_object(kind: str, type_id: int) -> bpy.types.Object | None:
    name = {"tree": TREE_OBJECTS, "bench": BENCH_OBJECTS}[kind].get(type_id)
    ...
```

`city_generator` 将 `_object_from_collection(..., idx)` 换为 `resolve_street_object(...)`；面板仍显示「类型1/2」，文档中写明每档对应的具体模型名。

**做法 B**：把团队 `.blend` 放进 `assets/Assets/Default/Road/`，更新 `All assets`，完全走 ICity 浏览器——与 Smart City 面板脱节，一般不优先。

### 7.4 与 `config/templates.json` 对齐

阶段二「应用模板」时，JSON 字段应与映射表一致，例如：

```json
{
  "0": { "tree_type": 1, "bench_type": 1, "road_texture": 2 },
  "1": { "tree_type": 2, "bench_type": 2, "road_texture": 1 }
}
```

`apply_template` 写入 `scene.scg_*` 后调用 `generate_base_city_from_scene()` 即可复用 Day 2-3 整条链路。

## 8. 常见问题

### Q1：类型2 一定是 Tree9 / Bench10 吗？

**不一定。** Day 2-3 只保证是 `ICity_Tree.objects[1]`、`ICity_Bench.objects[1]`。是否与 `Road assets.blend` 里某个命名资产相同，取决于 Start 模板里集合的 **排列顺序**。用 §4 脚本确认。

### Q2：为什么 ICity 面板能看到很多 Bench，我们只有类型1/2？

ICity 使用完整资产索引 + 按名 append；Smart City 为作业流程做了 **简化枚举**，阶段二可通过显式映射扩展含义，而不必一次暴露全部 Bench。

### Q3：放了 team 贴图仍不变？

确认路径为 `icity/assets/team/textures/road_type1.jpg`（扩展名、大小写），重载插件后重新 **生成基础城市**；贴图不存在时代码会跳过且不报错。

### Q4：团队新做的树木 .blend 放哪？

推荐 `assets/team/models/your_tree.blend`，阶段二在 `assets_manager` 登记路径与 Object 名，生成前 `wm.append` 并写入 `Road 2` 的 `Tree` 节点（或先放入 `ICity_Tree` 再按索引/名称引用）。
