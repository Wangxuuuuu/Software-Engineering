# Day 2-3：「生成基础城市」功能说明

## 与 Start / Edit 的区别

| 操作 | 作用 |
|------|------|
| **Start** | 载入 ICity 引擎（集合、ICity Base、几何节点） |
| **Edit City** | 进入 `ICity Base` 编辑模式 |
| **生成基础城市** | Start（若需要）+ **按面板参数批量写 Attribute** + 道路节点 + Sync + 再 Edit |

## 面板参数映射

| 参数 | 写入位置 |
|------|----------|
| 城市规模 小型 | 约一半面 `space type=Park`，一半 `Procedural` |
| 城市规模 中型 | 全部面 `space type=Procedural` |
| 建筑密度 低/中/高 | `Floor count` / `Floor count max` |
| 建筑样式 类型1/2 | `Procedural index`（0/1） |
| 道路类型 类型1/2 | 团队贴图 `assets/team/textures/road_typeN.jpg`（若存在）→ Road 材质 |
| 树木/座椅 类型1/2 | 边 Attribute + `Road 2` 节点 Object 引用（详见 [资产机制说明.md](./资产机制说明.md)） |

## 常见问题（验证 Day 2-3）

### 参数旁的圆点 / 菱形？

Blender 在 `use_property_split=True` 时对 **Enum（单选）** 的默认样式：圆点表示未选中项，菱形表示当前选中项，选中行会高亮。这是正常 UI，不是额外功能。

### 连点两次「生成基础城市」会进出 Edit？

**旧版**在生成末尾调用了 `sna.edit_city_d7cab`，该 Operator 会 **toggle** 编辑模式（在 Edit 时会退出）。  
**现已修复**：生成在 **Object 模式** 写 Attribute 并刷新；默认 **不再** 自动 Edit。若需要可勾选 **「生成后进入编辑」**（使用不 toggle 的 `enter_icity_edit_mode`）。

### 改了参数但视口不变，再点一次才变？

多与 **在 Edit 模式下写网格** 或 **未刷新 depsgraph** 有关。  
**现已修复**：生成前强制 Object 模式 + `_flush_viewport_refresh()` + 两次 Sync。

### 退出 Edit 会影响批量配置吗？

**不会**。配置在点击「生成基础城市」瞬间已完成（写 Attribute、改道路节点）。  
退出 Edit 只影响你是否手动画线/选面，不影响已写入的数据。

## 验证步骤

1. 重载插件 → Start（若尚未初始化）
2. Smart City 面板设置参数 → **生成基础城市**（保持 Object 模式即可看到变化）
3. 视口应出现建筑/道路变化；信息栏显示参数摘要
4. 需要手调时再点 **Edit City** 或勾选「生成后进入编辑」
