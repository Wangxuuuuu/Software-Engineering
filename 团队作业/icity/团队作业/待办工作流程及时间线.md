## 1. 工作目标与验收底线

### 1.1 核心定位

Web 主系统可以是 HTML 原型；**Blender 插件必须实现真实 3D 功能**。插件侧承担：

- 城市场景的生成与编辑
- 2D/3D 资产加载与替换
- 模板化一键配置
- 自然语言/JSON 命令执行
- 动态元素与生态场景
- 布局控制
- 渲染图片/视频输出

### 1.2 必须通过的验收项（§八 + §九）

| 编号 | 功能 | 通过标准 | 优先级 |
|------|------|----------|--------|
| A1 | 插件安装 | Blender 启用后侧边栏面板正常显示 | P0 |
| A2 | 基础生成 | 点击按钮后出现道路、建筑、树木等 | P0 |
| A3 | 2D 资产扩充 | **新增**至少 1 种道路纹理，替换后肉眼可见变化 | P0 |
| A4 | 3D 资产扩充 | **新增**至少 1 种路灯，能正确加载到场景 | P0 |
| A5 | 模板 0 | 输入 `0` 自动设置树木、道路、座椅类型 | P0 |
| A6 | 自然语言 | ≥ **5 条**中文指令稳定触发，无报错 | P1 |
| A7 | 动态元素 | ≥ 1 种（车/人）沿合理路径移动 | P1 |
| A8 | 生态元素 | 水体/山体/船只 ≥ 1；最好有船只移动 | P1 |
| A9 | 布局控制 | 输入点集+边集后道路布局发生变化 | P1 |
| A10 | 渲染输出 | 生成可打开的图片或视频 | P0 |

### 1.3 面板六区域目标（§4.1）

## 10. 风险与缓冲策略
