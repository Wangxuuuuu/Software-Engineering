## 3. 阶段二关键决策（复盘）

1. **模板不替换路灯**：应用模板统一 ICity 默认路灯；团队路灯仅区域 3。
2. **templates.json 去掉 `street_light`**：路灯不由模板 JSON 控制。
3. **面板区域 4 命名**：「模版选择」（主文档原「区域 2」与道路区冲突已调整）。
4. **视口**：道路/路灯/模板对比均建议材质预览或渲染。

## 4. 组内协作问答（Day 9 整理）

### Q1：LLM 与 templates.json

- **保持当前 JSON 即可**作为模板 0/1 的权威配置。
- LLM **不负责改** templates.json 来实现自然语言；阶段四用 **commands JSON** 联调。
- 若新增模板或改字段，由后端改 JSON 并通知 LLM / 前端。

### Q2：前端与 templates.json

- **需要模板内容**做 Web「模板库」展示（字段说明 + 中文文案）。
- 可直接引用仓库内 `config/templates.json` 或维护并行展示表；与 NL 命令协议无关。

详见：[`templates与组内协作说明.md`](templates与组内协作说明.md)

## 5. GitHub 同步说明（给全组）

上传 **`icity` 整个文件夹** 即可（含 `assets/team/`、`config/`、`团队作业/`）。

**建议 `.gitignore`**（可选）：`.idea/`、`.vscode/`、`__pycache__/`、本地 `test.py` 实验输出。

**新人上手顺序**：

1. 读根目录 `README.md` → 安装启用插件  
2. 读 `团队作业/待办工作流程及时间线.md` 了解全局  
3. 按分工读 `Day 9/templates与组内协作说明.md` 或具体 Day 记录  
4. 开发时只改 `scg_*` / `core/` / `operators/`，避免大改 `__init__.py` 内 ICity 生成代码
